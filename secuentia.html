DIAGRAMA DE SECUENCIA POR ROLES

MÓDULO: Autenticación (Login / Olvidó Contraseña)
Usuario (Cualquier Rol)
Vista / Frontend (`login.html`)
Controlador (`authController.js`)
Base de Datos (Supabase)
1.1

El usuario ingresa credenciales en login.html o solicita recuperación en forgot-password.html.

1.2

El formulario envía los datos via POST a authController.js.

1.3

El controlador consulta la tabla USUARIO, verifica contrasena_hash y extrae el id_rol.

1.4

El servidor responde redireccionando al usuario a su respectivo Dashboard según su nivel de acceso.

Flujos y Limitaciones de Casos de Uso por Rol

ID [1]: Administrador Acceso Total

Vistas asignadas: usuarios.html, juntas.html, index.html

Acciones Permitidas (Secuencia):
  • Gestionar usuarios globales y asignar/modificar el id_rol.
  • Crear y editar Juntas Vecinales en la tabla JUNTA_VECINAL.
  • Auditar el funcionamiento general del sistema.
ID [2]: Vecino Limitado a Barrio

Vistas asignadas: juntas.html, reportes.html

Acciones Permitidas (Secuencia):
  • Visualizar información de su propia junta vecinal asociada (id_junta).
  • Crear nuevos reportes ciudadanos (Tabla REPORTE).
  • Recibir alertas de seguridad generadas en su zona geográfica mediante notificaciones.html.
ID [3]: Policía Operativo / Respuesta

Vistas asignadas: incidentes.html, alertas.html, patrullajes.html, analis.html

Acciones Permitidas (Secuencia):
  • Revisar incidentes reportados y registrar la validación en VALIDACION_INCIDENTE.
  • Emitir alertas comunitarias inmediatas de alto riesgo en la tabla ALERTA.
  • Visualizar mapas críticos y mapas de calor (mapaController.js).
  • Registrar bitácoras de rutas de vigilancia en la tabla PATRULLAJE.
ID [4]: Moderador Filtro de Contenido

Vistas asignadas: reportes.html, configuracion.html

Acciones Permitidas (Secuencia):
  • Evaluar la veracidad de los reportes enviados por los vecinos para evitar falsas alarmas.
  • Aprobar o rechazar el reporte modificando el campo estado en la tabla REPORTE.
  • Escalar reportes aprobados transformándolos formalmente en filas de la tabla INCIDENTE.

Trazabilidad del Sistema:

Este flujo HTML vincula de manera lógica la estructura del backend de NodeJS expresada en tus controladores (authController.js, incidentesController.js, juntasController.js) con los estados de persistencia de datos relacionales en tu base de datos de Seguridad Ciudadana.
